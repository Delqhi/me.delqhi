# Docs index

## What this is
Navigation for the repo's documentation tree.

## What it's for
Keeps the standard repo layout discoverable for humans and agents.

## How to update
Add a new section here whenever you add a new docs folder or top-level doc.

## Structure
- `00_vision/` — why the repo exists and where it is going
- `01_design/` — architecture, terminology, and decision records
- `02_spec/` — product and interface specifications
- `03_ops/` — runbooks and infrastructure notes
- `04_legal/` — compliance, security, and governance
