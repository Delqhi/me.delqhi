# A2A-CARD.md

## What this is
A simple capability card for other agents.

## What it's for
Lets an external agent understand the repo scope without reading the whole app.

## Capabilities
- Portfolio and landing-page UI
- Blog index and blog detail routes
- Repository-style projects view
- README-style in-app documentation page

## Boundaries
- No public backend API today
- No database or persistence layer
- No secret values committed to the repo

## How to update
Update this card when routes, content models, or deployment assumptions change.
