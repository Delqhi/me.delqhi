# Governance

## What this is
Ownership and change-management notes for the repository.

## What it's for
Makes it clear who maintains the site and how documentation should stay in sync.

## How to create or update
Update this file when ownership, review rules, or release flow changes.

## Current governance
- Jeremy Schulze owns the site content and direction.
- Functional changes should update the matching docs page.
- Architectural changes should create or update an ADR.
- Release candidates should pass lint, build, and tests before merge.

## Best practices
- Keep decisions in ADRs instead of burying them in comments.
- Keep docs changes small and tied to the code change they describe.
